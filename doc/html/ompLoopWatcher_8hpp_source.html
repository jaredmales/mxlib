18 ///A class to track the number of iterations in an OMP parallelized loop.
19 /** Uses omp critical directives to deconflict updates by different loops. Example:
20  * \code
21  ompLoopWatcher<> watcher(1000, std::cout); //Uses defaults
22 
23  #pragma omp parallel for
24  int( i=0; i<1000; ++i)
25  {
26  watcher.incrementAndOutputStatus();
27  //Do loop work
28  ...
29  }
30  \endcode
31  * This will result in the following output
32  \verbatim
33  1 / 1000 (0.1%)
34  2 / 1000 (0.2%)
35  3 / 1000 (0.3%)
36  \endverbatim
37  * and so on.
38  *
39  * \note This can reduce performance due to critical points it creates if used in fast loops (e.g. do not use inside the inner most loop!).
40  *
41  * The behavior of the output is controlled through template parameters. A different output-type can be specified, which needs to accept size_t, and optionally
42  * float, and character input using the << operator.
43  *
44  * \tparam outputT a type which accepts size_t, float, and character input via the << operator (default is std::ostream).
45  * \tparam _printPretty flag to control whether the output is nicely formatted, if false then just the numbers are sent to the output with no spaces or delimiters (default is true).
46  * \tparam _printLoops flag to control whether the number of loops is sent to the output each time (default is true).
47  * \tparam _printPercent flag to control whether the percentage complete is calculated (default is true).
48  * \tparam _printNLine flag to control whether the newline '\\n' is sent ot output (default is true).
49  *
50  * \ingroup mtutils
51  */
