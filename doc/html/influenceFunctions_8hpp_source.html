354 ///Calculate the pseudo-inverse and mirror modes for a set of influence functions.
355 /** The influence functions are a cube of 2D maps of mirror deformations, and the matrix \f$ A \f$ is formed by vectorizing the influence functions.
356  * The pseudo inverse is then calculated from the SVD as follows:
357  * \f$ A = U S V^T \f$
358  * which gives the Moore-Penrose pseudo-inverse:
359  * \f$ A^+ = V S^+ U^T\f$.
360  * The columns of U contain the orthogonal mirror modes, and S contains the singular values. These are both written to disk along with the pseudo-inverse.
361  *
362  * \param[in] dmName is the name of the deformable mirror.
363  * \param[in] maxCondition [optional] the maximum condition number to accept for the pseudo-inverse. If < 0 [default] then the user is interactively asked what to use.
364  *
365  * \tparam realT is the real floating point type in which to do calculations.
366  */

434 ///Calculate the modes-to-commands matrix for a set of modes
435 /**
436  * Given the pseudo-inverse \f$ A^+ \f$ of the influence functions, the commands to the actuators to take a mirror shape \f$ \vec{s} \f$ are calculated as
437  * \f[
438  \vec{c} = \mathbf{A^+} \vec{s}.
439  * \f]
440  * Now given a basis set \f$ \mathbf{M} \f$, the mirror shape is specified as
441  \f[
442  \vec{s}= \sum_i m_i M_i
443  \f]
444  * where \f$ \vec{m} = m_0, m_1, \cdot m_i \cdot \f$ are the modal coefficients. If the mirror-to-commands matrix, \f$ \mathbf{M2c} \f$ gives commands as
445  * \f[
446  \vec{c} = \mathbf{M2c} \vec{m}
447  * \f]
448  * then we can calculate \f$ \mathbf{M2c} \f$ as:
449  * \f[
450  \mathbf{M2c} = \mathbf{A}^{+T} \mathbf{M}
451  \f]
452  *
453  * \param[out] M2c is the M2c matrix, allocated during the calculationg
454  * \param[in] Ap is the pseudo-inverse of the influence function basis set
455  * \param[in] M is the modal basis set, a cube of shapes
456  *
457  * \tparam realT is the real floating point type used for all calculations.
458  */
459 template<typename realT>
460 void m2cMatrix( Eigen::Array<realT, -1, -1> & M2c,
461  Eigen::Array<realT, -1, -1> & Ap,
462  mx::eigenCube<realT> & M )
463 {
464  M2c = Ap.matrix().transpose() * M.asVectors().matrix();
465 }
